# Tag Lens 基本設計書 (v0.3.12)

Obsidian コミュニティプラグイン。vault 内ノートの**タグ集合関係**を多様な図で可視化し、
さらに分析・タグ整理・属性ベースの視覚エンコーディングを提供する。

- 形態: 単一の `ItemView`（タブ `tag-lens-view`）内の **HTML5 Canvas (2D)** に描画。
- 制約: `minAppVersion 1.5.0` / デスクトップ+モバイル / **完全ローカル（ネットワーク通信なし）**。
- ビルド: TypeScript + esbuild → `main.js`。型検査は別途 `tsc --noEmit`（`npm run verify`）。

---

## 1. レイヤアーキテクチャ（データフロー）

```
Vault (Markdown)
   │  metadataCache（tags / links / frontmatter / mtime）
   ▼
[parser]  buildGraph        … ノード生成（GraphNode: tags=memberships, links/backlinks, fmStatus/fmMaturity/ageDays）
   ▼
[query]   WHERE/GROUP_BY/HAVING/ORDER_BY/LIMIT  … 「どのノート/クラスタを出すか」= データ選択層
   │       （filterMode: "sql" 内蔵エンジン or "dvjs" DataviewJS）
   ▼
[layout]  layout()  → mode別レイアウト関数 … 図形配置（PositionedNode / LaidOut）
   ▼
[encoding] evaluateEncoding … 表示中ノードの属性→視覚チャネル（色等）。**表示集合は変えない** = 視覚層
   ▼
[draw]    draw() → mode別 draw-*.ts … Canvas へ描画
   ▲
[UI panel] 統合コントロールパネル（Filter / Notes / Settings(View/Display/Encode/Layers) / Insight）
```

**最重要の設計原則：データ選択層（query, SQL/dvjs）と視覚層（encoding）は別目的・別レイヤー。**
Encoding は「どのノートを表示するか」を一切変えず、各ノードの描画パラメタのみ与える。

## 2. View Mode（11種）

| id | 名称 | 概要 | 種別 |
|---|---|---|---|
| `droste` | Icon Gallery | ノートごとのアイコン（同一/部分共有タグ群＋リンク/被リンクを入れ子）をタイル表示 | 単一ダイアグラム |
| `lattice` | Intersection lattice | 次数階層の Hasse 風レイアウト＋部分集合リンク。count/zoom で overview/density/individual 自動切替 | 画面空間 |
| `heatmap` | Co-occurrence heatmap | tag×tag 共起グリッド（Jaccard or log+p95 カウント）。Gap 可視化対応 | 画面空間 |
| `upset` | UpSet plot | 交差シグネチャ別カード＋ドット行列（≥4-way 対応） | ハイブリッド |
| `matrix` | Connection matrix | notes×tags ドット行列（seriation） | 画面空間（experimental） |
| `bipartite` | Tag graph | notes+tag ノードの二部グラフ（力学/同心/クラスタ配置） | カード（experimental） |
| `bubblesets` | BubbleSets | 矩形 iso-contour の包含レイアウト | カード |
| `euler` / `euler-true` / `euler-venn` | Nested set / Containment map / Euler | 包含/重なり矩形系（experimental） | カード |
| `stream` | Sequence Stream | タグ×時間/値ビンのストリーム（休眠スレッド検出） | 画面空間（beta） |

> 「カードモード」= drawBodyTile/drawCard で world-space カードを描く群（euler系/bipartite/bubblesets/upset）。
> overlay（status色/freshness/maturity/encoding色）はカードモードで効く。

## 3. 主要機能

- **統合コントロールパネル**：浮遊パネル（移動/リサイズ/最小化/右ドック可）。タブ = Filter / Notes(navigator) / Settings / Insight。
- **Filter**：SQL内蔵式（WHERE/GROUP_BY/HAVING/ORDER_BY/LIMIT）または DataviewJS。
- **Display**：showNodes/Enclosures/Edges/Grid、minFontPx、freshness/status overlay、maturity バッジ。
- **Encode**：Visual Encoding Engine（属性→視覚チャネル束縛。第1弾 Color）。
- **Insight**：Cognitive Load 指標 / Alerts（Gap finder, Bridge finder, Stalled cluster, Ripening backlog）/ Suggest（タグ分類提案）。
- **Saved Lenses**：filter/表示設定のプリセット保存・適用（コマンド化）。
- **Active Note View**：アクティブノート追従でその文脈（リンク/被リンク/共有タグ）を表示。
- **PNG エクスポート**：全モードを高解像度でローカル保存/クリップボード。
- **Note navigator**：フォルダ/タグツリー、検索、表示/非表示トグル。

## 5. 主要データ構造（概要）
- `GraphNode`：id(path), label, memberships(tags), mtime, fmStatus, fmMaturity, ageDays。
- `PositionedNode`：GraphNode + x/y/width/height（レイアウト後）。
- `LaidOut`：nodes/edges/clusters + モード別メタ（upset/matrix/heatmap/lattice/stream/drosteGallery）。
- `MiniSettings`：全永続設定（viewMode, where/groupBy/having/..., 各モード設定, statusField/statusColors, freshnessOverlay/staleDays, showMaturity, **encoding: EncodingBinding[]** 等）。
- `EncodingBinding`：{ channelId, fieldId, scale, enabled }（視覚エンコーディング）。

---

## 6. 設計原則（不変条件）
1. **データ選択 ⊥ 視覚エンコーディング**：encoding は表示ノード集合を変えない。
2. **拡張はレジストリ追加で**：encoding のチャネル/フィールドは `register*()` 1エントリで増やせる。
3. **検証ゲート**：`npm run verify`（tsc&&test&&build）緑をマージ条件。esbuild は型を無視するため tsc が型の唯一のゲート。
4. **リファクタは挙動完全保存＋verify緑＋1抽出1コミット**（view.ts 分割: `refactor-view-split.md` 参照）。

詳細は同ディレクトリ **詳細設計書.md** / **AGENTS.md** を参照。
