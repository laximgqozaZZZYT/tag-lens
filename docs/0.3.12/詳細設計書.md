# Tag Lens 詳細設計書 (v0.3.12)

基本設計書の各レイヤをモジュール単位で詳述する。行番号は god-file で頻繁にずれるため、
着手時は `grep -n`（`layout.ts` は **`grep -a`**）でアンカーを取り直すこと。

---

## 1. データ層

### parser.ts — `buildGraph(app, whereRows, groupByRows, filterMode, dvjsFilter, statusField)`
- 全 Markdown を走査し `GraphNode[]` を生成。
- 各ノード: `id=path`, `label=basename`, `memberships`(GROUP_BY 由来のクラスタキー), `mtime`,
  `fmStatus`（`statusField` の frontmatter 値を lowercase）, `fmMaturity`（後述）, `ageDays`。
- **maturity**: frontmatter `maturity` を `effectiveMaturity(persisted, suggestMaturity(...))` で解決
  （妥当値は override、不正/欠落はヒューリスティック）。backlink 数は事前計算。
- links/backlinks は cache.links + frontmatterLinks。

### query.ts / query-pipeline.ts / query-filters.ts / qp-1d.ts
- `parseQuery` / `evalQuery` / `isMatched`、`FileFacts{path, tags, frontmatter, tagProperties}`。
- WHERE/GROUP_BY/HAVING/ORDER_BY/LIMIT。`tagN:` 階層、`tag.<key>:` join、AND/OR/XOR/NOR/NAND/glob/fuzzy。
- `havingMode: "filter"|"highlight"`（filter=落とす / highlight=残して強調）。
- **filterMode**: `"sql"`（内蔵）/`"dvjs"`（DataviewJS `dv.pages()` の出力をノード集合に）。

---

## 2. レイアウト層

### layout.ts — `layout(layoutData, sized, opts): LaidOut`
- `opts.viewMode` で分岐（`grep -a` 必須）:
  - `euler-true`→`layoutEulerTrue`（bubblesets も流用）, `euler-venn`→`layoutEulerVenn`, それ以外→`layoutEulerNested`。
  - upset/matrix/heatmap/lattice/bipartite/stream は各 `*-layout.ts`。
- **PositionedNode への属性伝播**：各 `nodes.push({...})` は `mtime/fmStatus/fmMaturity/ageDays` を
  必ず引き継ぐこと（過去 euler-true/venn で欠落し overlay が無言で効かないバグがあった）。
- `LaidOut`：`nodes/edges/clusters` + 任意 `upset/matrix/heatmap/lattice/drosteGallery/stream/setNodeIds`。
  - `setNodeIds`（bipartite の SET ノード集合）が立つとカードグリッドは抑止。

### 補助
`cluster-bbox.ts`（クラスタ bbox/継承）, `anchor-placement.ts`, `edge-routing.ts`, `subgroup-*.ts`,
`region-layout.ts`（euler系: 旧スペックは `docs/old/region-layout-spec.md`）, `*-layout.ts` 各種。

## 3. 描画層

### view.ts `draw()` — モード別 dispatch
- matrix/heatmap/lattice/droste/stream は専用 `draw-*.ts` を呼んで return（画面空間 or 単一ダイアグラム）。
- カード系は world-map タイル経由で `drawBodyTile`→`drawCard`、enclosure/grid/edges を描く。
- **モード別ガード**（適用表 `display-applicability.ts` と一致させること）:
  - `showEnclosures`/`showEdges`/`showGrid` は `!this.laid.upset`（upset では非描画）。
  - grid は `!this.laid.setNodeIds`（bipartite では非描画）。

### draw-card.ts `drawCard(ctx, n, opts)` — 純粋カードレンダラ
- fill 優先順位: highlighted → SET(`fillHue`) → tint → **`encFillColor`**（Visual Encoding 色）→ 既定 `canvasBgAlt`。
  - `encFillColor` は**additive**（未指定なら従来通り）。
- `statusColor`（枠線, `!isSet`）, freshness `globalAlpha=freshnessAlpha(mtime,now,staleDays)`, maturity バッジ。
- 数値→色は `clusterHue`(0..360) を `hsl()`/`theme().swatch` で文字列化（**数値を直接 fillStyle に入れない**）。

### draw-stream.ts ほか
- フォントは `Math.max(floorFontPx, …)` で minFontPx を尊重。screen-space モードは `canvas.width/dpr` を使用。

## 4. Visual Encoding（src/encoding/）— 第1弾: Color

### 責務（不可侵）
属性→視覚チャネルの写像のみ。**表示ノード集合を変えない**（query/dvjs と別レイヤー）。

### types.ts
- `FieldSource{id,label,kind(categorical|quantitative|temporal|ordinal),accessor(node,ctx)}`
- `VisualChannel{id,label,accepts,appliesTo(mode),apply(params,scaled,ctx)}`
- `EncodingBinding{channelId,fieldId,scale?,enabled}`（= `MiniSettings.encoding[]`）
- `ScaleConfig{type,domain?,palette?,reverse?,clampPctl?}` / `ScaledValue{t?,category?,output?,missing?}`
- `NodeDrawParams{fillColor?,fillHue?,sizeScale?,opacity?,icon?,borderColor?,groupKey?,axisX?,axisY?,label?}`
- `EncContext{nowMs, degreeOf?, frontmatterOf?}`（Obsidian 依存ルックアップを隔離）

### field-sources.ts（レジストリ）
- `registerFieldSource()` / `resolveFieldSource(id)`（動的 `frontmatter:<key>` を `frontmatterField()` で生成）。
- 組込み: status / maturity / ageDays / tag / degree / inDegree / outDegree。

### channels.ts（レジストリ）
- `registerChannel()` / `resolveChannel(id)`。組込み = `color`（`appliesTo:()=>true`、scaled→`fillColor`）。

### scales.ts — `prepareScale(config, rawValues) → {apply, legend}`
- quantitative: linear/log/quantile、自動 domain・p95 クランプ・reverse → `{t:0..1}`。
- categorical/ordinal: palette 上書き or `autoColor`(clusterHue) → `{category, output:hsl}`。`legend` 同時生成。

### evaluate.ts — `evaluateEncoding(nodes, bindings, ctx, mode?) → {params: Map<id,NodeDrawParams>, legends}`
- enabled binding ごと: ドメイン推定→ノード毎 `accessor→scale→channel.apply`。
- **不変条件（テスト固定）**: 入力ノードを mutate しない / `params.size == nodes.length`（選択非干渉）。
- 未知 field/channel は黙ってスキップ。

### migrate.ts
- `synthesizeEncodingFromLegacy(settings)`（status overlay→color binding）/ `effectiveEncoding(encoding, legacy)`。

### view.ts 配線
- `rebuild()` 末尾で `EncContext{nowMs:Date.now(), degreeOf:degreeMap, frontmatterOf:metadataCache}` を組み、
  `evaluateEncoding(this.laid.nodes, settings.encoding, ctx, viewMode)` → `this.encParams/this.encLegends`。
- `drawCard` opts に `encFillColor: encParams.get(n.id)?.fillColor`。
- 「Encode」タブ `renderSettingsEncode`（color 行: フィールド束縛/スケール/reverse/自動凡例）。

### テスト
`test/encoding-{scales,evaluate,migrate}.test.ts`（純粋・Obsidian 非依存）。

## 5. Insight（src/insight/）
- `compute.ts`：`computeCognitiveLoad(k)`（可視 node/edge/cluster から認知負荷スコア）、`computeTagSuggestions`
  （タグ別 count/ratio + パーセンタイル+エントロピーで Golder 分類提案）。**ほぼ純粋**。
- `render.ts`：`renderInsightOverview/Alerts/Suggest`（DOM 構築。Alerts は Gap/Bridge/Stalled cluster/Ripening backlog）。
- `actions.ts`：`applyGolderClassification`（tagページ frontmatter `golder_type` 書込）、`convertToNestedTag`。

## 6. パネル UI（src/panel/）
- `panel-sections.ts`：`renderToggleSection`/`renderOrderBySection` 等（deps `{settings,save,redraw}` 受取の自由関数）。
- `panel/settings-sections.ts`・`panel/settings-tabs.ts`：view.ts から抽出した設定 UI（同 deps パターン）。
  view.ts 側は薄い委譲。さらなる分割計画は `refactor-view-split.md`。
- `note-menu.ts`：navigator（フォルダ/タグツリー、検索、表示トグル、タグページ操作）。

## 7. 設定・型・適用判定
- `types.ts`：`MiniSettings`（全永続設定）/ `DEFAULT_SETTINGS` / `ViewMode` / `VIEW_MODES` / `GraphNode` /
  `EncodingBinding`。新フィールド追加時は **interface と DEFAULT_SETTINGS の両方**を更新（過去の型崩れ要因）。
- `display-applicability.ts`：`displayToggleApplies(mode,key)`（Display トグルのモード別出し分け。実描画ガードと一致必須）。

## 8. ハマりどころ（再掲・AGENTS.md と同期）
1. `src/layout.ts` は NULバイト混入 → 検索は **`grep -a`**（素の grep は無警告で空を返す）。
2. `src/view.ts` は ~4800行の god-file。編集前に `grep -n` でアンカー再取得。
3. `npm run build`/`npm test` は型を検査しない → **`tsc --noEmit`（`npm run verify`）が唯一の型ゲート**。
4. Encoding は表示集合を変えない。レイアウトの属性伝播漏れは overlay の無言失敗を生む。
5. E2E は「例外なし」で合格としない → **実際の反映**（draw params / laid.nodes 値）を検査。別プロファイル+専用ポート+後始末必須。
