> **このドキュメントが唯一の正。** 経緯: 等角写像 `exp(γζ)` の渦・赤格子方式を断念 →
> 直交ドロステ ½再帰＋ズームトンネル方式 → **2026-05-31 にドロステ再帰も廃止**。
> 再帰は (a) 自己言及にならず (b) 最上段しか読めず (c) グラフの分岐を1本の鎖へ潰す、
> という理由で割に合わなかった（kaizen 分析）。価値ある **①②③④ 包含ユニットだけ残し**、
> 1段を画面いっぱいに描く「Containment Lens」に確定。内部 ID は設定互換のため `droste` 据え置き。

## §0. 何を作るか（一文）

**焦点ノード N の所属タグ包含構造 ①②③④ を、直交座標の同心入れ子として1段だけ大きく描くレンズ。**
別ノードへの移動は「図中のノードをクリック」または「フォーカスピッカー（検索＋ツリー）」で
焦点 N を切り替え、その N を中心に再描画する。ズーム／再帰／ズームトンネルは無い。

### §0.1 体験（操作と見え方）
- ズームイン → 現在の焦点の枠が画面外へ広がり、中心の縮小コピー（次の焦点）が実寸大に。
  その中心にさらに次が現れる。1段ぶん（×2）ズームするごとに焦点が1つ繰り上がる。
- ズームアウト → 逆に前の焦点へ戻る。
- 各焦点で、そのノードを N とした ①②③④ が展開される。
- 関連を辿って潜るので、ズームは vault を貫く「関連のトンネル」になる。

- 等角写像・赤格子メッシュ・渦・seg分割・copies周回・bbox→ζ巻き取りは**全て廃止**。
- `conformal.ts` は**不要**（削除または未使用化）。
- 自己相似 = 渦ではなく「正方形の中に同じ正方形が入れ子」で表現。

### §0.1 破棄した方針（再びこちらへ逸れない）
- ❌ 等角写像 `z=R₀·exp(γζ)` による渦・対数螺旋
- ❌ 赤格子メッシュ（一様グリッドの頂点写像）
- ❌ strip 巻き取り（v=角度, u=半径）、copies の周回重ね描き
- ❌ 自前タイル生成・差分スライス・可変帯充填（旧方針の残骸）

## §1. 入れ子の単位（1段ぶん）

焦点ノード N の所属タグ積集合を `T = N.memberships` とする。
1段の入れ子を**同心の矩形**として描く（外→内）:

- **④（最外枠）**: `T` の**真部分集合**を持つグループ（囲い）。例 `act`, `drama`。`|signature|`降順
- **③**: 積集合 `T` の交差グループ（囲い）。例 `act ∩ drama`
- **②**: `T` と**完全一致**するノード群を囲う枠（N の真の兄弟。部分一致は除外）
- **① N**: 中心の特定ノード1個（＋②の兄弟ノードのセルを周囲に配置）
- （⑤「その他ノード」は廃止。ノードの探索・選択は §3 のフォーカスピッカー（ツリー＋検索）で行う。）

包含: `① ∈ ② ∈ ③ ∈ ④`。データは `node.memberships` の集合演算で算出
（② = `setEqual(memberships, T)`、④ = `signature ⊊ T`、実在 signature に限定）。

## §2. ドロステ再帰（中心へ ×1/2、焦点を次ノードへ送る）＋ ズーム

各段の中心コピーは**同じ N ではなく「次の焦点」**にする（同一コピーの縮小は無意味だった）。
次の焦点 = **現在の焦点に最も関連が強い未訪問ノード**（関連 = memberships の積集合サイズが
最大。同数はファイル名順で決定的に）。関連の未訪問が尽きたら vault 全体の未訪問から
固定順序で拾う（全ノード到達の担保）。

```
chain(focus0):                          # 焦点の訪問列を作る
    visited = {}; seq = []; f = focus0
    while f and len(seq) < maxDepthCap:
        seq.append(f); visited.add(f)
        f = mostRelatedUnvisited(f, visited)   # 関連最強の未訪問
            or anyUnvisited(visited, order)     # 尽きたら全体順で次
    return seq

drawNest(rect, seq, i, depth):
    if depth == 0 or rect < minFontPx or i >= len(seq): return
    drawUnit(rect, seq[i])                       # seq[i] を焦点に ①②③④（同心入れ子）
    inner = scaleAboutCenter(rect, 1/2)
    drawNest(inner, seq, i+1, depth-1)           # 中心は次の焦点 seq[i+1]
```

- `k=0.5`、`depth=5`（一度に見える段数）、描画順は外段→内段。
- **ズーム**: ホイール/ピンチで中心へズーム。倍率が ×2 進むごとに表示の起点インデックス i を
  +1（焦点を1段繰り上げ）。シームレスに次の焦点が実寸大になり、新たな中心コピーが湧く。
  ズームアウトは i を戻す。これにより「ズームだけで seq 全体（=全ノード）を辿れる」。
- ⑤（その他ノード）は廃止済み。ズームは右上ツールバーの 🔍 in/out（またはホイール）で操作。

## §3. インタラクション（再ルート）

- **図中のノードを押下 → そのノードを新しい焦点 N とし、`T` を再計算して全体を再描画。**
  これがこのビューの主操作。潜る楽しさ＝クリックで焦点が変わり、その N を中心に入れ子が組み直る。
- ホバー: ノードのツールチップ。空き領域クリック: 無視。
- ヒットテスト: 直交座標なので素直な矩形内包含判定。複数段が重なる中心付近は
  **最前面（最内段＝最も小さく描かれた段）**を優先。
- **フォーカスピッカー（ミニメニュー）**: ツールバーの list アイコンで開閉。検索欄＋
  ノードのツリー（パス階層）を持ち、選択したノードを新しい焦点 N にする（⑤廃止に伴う
  ノード探索手段）。

## §4. レンダリング

- ソース幾何 = `laid.nodes`（カード矩形）/ `laid.clusters[].pieces[]`（囲い矩形・包含構造）/
  `laid.edges`。BubbleSets レンダラ本体は触らず、論理レイアウトを列挙して矩形描画。
- 全て直交座標の矩形＋テキスト。歪みなし。ラベルは水平、セル幅クランプ＋"…"、minFontPx 未満非表示。
- 色: 既存 clusterHue を流用（④の枠色、N のハイライト等）。

## §5. 設定

| フィールド | 既定 | 意味 |
|---|---|---|
| drosteScale (k) | 0.5 | 1段ごとの縮小率（中心コピーの倍率） |
| drosteDepth | 5 | 再帰段数 |
| drosteFocus | "" | 焦点ノードID（空=先頭ノード、クリックで変更） |

（旧 drosteZoom/twistDir/copies/subdiv は廃止）

## §6. 検証

- 直交プロットで `① ∈ ② ∈ ③ ∈ ④` の同心入れ子が出る（確認済み）。
- その中心に 1/2 縮小コピーが5段、マトリョーシカ状に入れ子になる。
- 中心付近は minFontPx 未満で自然に省略され、無限縮小の見た目になる。
- ノード押下で焦点が変わり再描画される。
- `npx tsc --noEmit` + `npm run build` + vault 目視。
