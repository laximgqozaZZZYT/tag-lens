# Containment lens — Icon Gallery 設計（確定版 2026-06-01）

> **このドキュメントが唯一の正。** 旧「焦点1段の包含レンズ」（2026-05-30）を置換する。
> VIEW MODE id は設定互換のため `droste` のまま。再帰ズーム/トンネルは廃止済み（再導入しない）。

## §0. 何を作るか（一文）

**vault に存在する全ノードについて、そのノードを中心とする「イコン画」を生成し、互いに
重ならないようギャラリー状に並べたビュー。** 常時表示のミニメニュー（ツリー＋検索）で
ノードを選ぶと、そのノードのイコン画を画面中心へフォーカスする。

## §1. イコン画の構成（1ノード分）

焦点ノード N のタグ集合（GROUP_BY による memberships、NONE 除外）を `sig(N)` とする。
内側→外側に同心の正方形で積み上げる:

- **① 対象ノード N**：中心に1個。サイズ **4×4** セル。
- **② N の所属する積集合**（= `sig(N)` = `T`）**とその配下ノード群**：①を囲む正方形フレーム。
  配下 = `T ⊆ sig(M)` のノート（N 自身を除く＝N の全タグを共有するノート）。各 **2×2** セル。
- **③ ②の積集合の「直接の上位集合」となる集合とその配下ノード群**：②を囲むフレーム。
  「直接の上位集合」= `T` の**真部分集合** `O`（|O|<|T|）。レベル `d = |T|−|O|`（d=1⇒③₁ … 外側へ）。
  - **`T` の真部分集合 `O` は、その領域が非空（`O ⊆ sig(M)` のノートが1件でも存在）すればボックスを描く。**
    例 T={adversary,character} で「adversary 単独」ノートが無くても、`{adversary}` の領域は非空
    （adversary を持つノートが存在）なので **`{adversary}` ボックスを描く**（lilith が adversary 集合にも
    属することを示す）。
  - **ボックス内メンバ（配下）= `O = T ∩ sig(M)` が `O` に完全一致するノート**（排他メンバ。例
    `{character}` のメンバ = character は持つが残りの T タグを持たないノート）。排他メンバが0でも
    （全て内側の ② 等に居る場合）枠＋ラベルは描く。各 **1×1** セル。
  - **|T|≤3（3集合の組合せ）までは真部分集合を完全に描画する。** |T|≥4 はベストエフォート
    （サイズ `|T|−1`, `|T|−2` の部分集合に限定可）。`O=∅`（無共有）は対象外。
  - 比較不能な ③同士（例 {adversary} と {character}）は重ねず、③帯の中で別区画に並置する。
- **⑤「link | backlink」**：最外周の囲い。N と**リンク／バックリンク関係**にあるノートを入れる。
  link = N が指す先（発リンク, source→target）、backlink = N を指す元（被リンク, target→source。
  link と重複するものは除外し各ノート1回）。**リンクかバックリンクかで色分け**し、ラベル
  「link | backlink」も同色。サイズ・グリッド整列・上限 `4^n−1`・外向きラベル等は ③ に準拠
  （各 1×1 セル）。リンクもバックリンクも無ければ ⑤ は描かない。

### 配置と入れ子
- ①を中心に置き、②は①を、③は②を囲む。③の繰り返しは直前の③を囲む（同心フレーム）。
- **同一レベルに複数の独立集合（分岐：例 `{a,b,c}`→`{a,b}`,`{a,c}`,`{b,c}`）がある場合**は、
  そのレベルのフレーム帯を**区画分割して並置**する（関係ない集合同士を重ねない）。
- すべて軸並行の正方形。集合の囲いのサイズは内容に応じて可変。

### 描画上限と集約
- レベル `n`（①=1, ②=2, 1回目の③=3, 2回目の③=4, …）のノード群描画上限は **4^n − 1**
  （②:15, ③₁:63, ③₂:255, …）。上限は**そのレベルの各集合（ボックス）ごと**に適用する
  （独立した集合がそれぞれ見えるように）。
- 上限超過分は**集約表示**（`+N` の小箱）にまとめる。

### 省略規則
- **領域が空の集合は描かない**（そのタグの組合せを持つノートが vault に1件も無い `O` は出さない）。
  ただし「領域は非空だが排他メンバ0」の集合は枠＋ラベルを描く（上記 ③）。
- **|T|≥4（タグ4個以上）の組合せはベストエフォート**（サイズ `|T|−1`,`|T|−2` に限定可）。同心入れ子と
  非重複を構造上両立できない分岐は省略してよい。

## §2. ギャラリー（全ノード配置）

- 存在する全ノードそれぞれにイコン画を1枚生成し、**等間隔グリッド**に配置（互いに非重複）。
  列数 = `ceil(sqrt(ノード数))`。セル = 固定サイズの正方形、内側にイコン画を収める。
- **パン/ズーム**で全体を回遊できる。描画は**ビューポート可視セルのみ**（カリング）。
  可視セルのイコン木はその場で構築（タグ数は小さく安価）。
- イコン画同士は重ならない（グリッドセルが分離）。

## §3. ミニメニュー（常時表示ポップアップ）

- ツリービュー（パス階層）＋検索欄を持つフローティングパネルを**常時表示**（トグルではない）。
  従来の右上トグルアイコンは廃止。
- ノードを押下 → そのノードのイコン画が**画面中心に来るようパン**（フォーカス）。
- 検索で絞り込み（部分一致、上限あり）、フォルダは開閉。現在フォーカス中ノードをハイライト。

## §4. インタラクション
- ギャラリー上のノード（① や配下セル）クリック → そのノートを開く＋（任意で）そのノードへフォーカス。
- ホイール=ズーム、ドラッグ=パン（他モードと同様）。ホバーでノート名ツールチップ。

## §5. データ／分類ロジック（受け入れ基準）
焦点 N、`T = sig(N)`、各ノート M の `sig(M)`:
- ② = `T ⊆ sig(M)` のノート（①の隣、2×2）。**該当なしでも②は空の枠として常に描く。**
- ③ = T の真部分集合 O（領域非空＝`O ⊆ sig(M)` のノートが存在）。ボックス内は排他メンバ
  `T ∩ sig(M) == O`。排他0でも領域非空なら枠＋ラベル（プレースホルダ）。
- 各集合（ボックス）ごとに **4^n − 1** で打ち切り、超過は `+N` 集約。
- 領域が空の集合（そのタグ組合せのノートが皆無）は非描画。`|T|≤3` は完全、`|T|≥4` はベストエフォート。

## §6. 影響ファイル
- `droste-layout.ts`：イコン木の生成（`buildIconDiagram(data, focusId)` … 世代レベルごとの
  集合・配下ノード・上限・集約）＋ギャラリーのグリッド座標算出。
- `draw-droste.ts`：同心フレーム＋ノード正方形＋集約箱の描画。可視カリング対応。
- `layout.ts`：droste 分岐は全ノードのグリッド配置（軽量）＋ drosteData を返す。
- `view.ts`：パン/ズーム有効化、常時表示ミニメニュー、メニュー選択→該当セルへパン、クリックで開く。
- `conformal.ts`：不要（不変）。

## §7. 非ゴール
- ズーム/再帰トンネルの再導入（廃止済み）。
- 忠実なオイラー図（独立集合が中心で必ず重なる表現）— 採らない。独立は区画分離。
